DCAT-AP-CH 2.0: Swiss Application Profile for OGD

This is the specification for Swiss Open Government Data. The Application that goes with this is opendata.swiss the OGD portal for Switzerland.

This is required.

Introduction

The next version of the DCAT-AP-CH seeks to overcome incompatiblities with the DCAT-AP. This specification is a draft to prepare for the next DCAT-AP-CH

Class specifications

Class: Catalog

In DCAT dcat:Catalog, dcat:Dataset and dcat:Distribution are all subclasse of a super-class dcat:Resource. Therefore they have many attributes in common The general rule is, that properties given on a higher level, such as the dcat:Catalog can be overwritten by those provided at a lower level, such as dcat:Dataset.

The following properties are specific to this class: dataset, homepage,

The following properties are mandatory in the DCAT-AP and also mandatory on the DCAT-AP-CH: themes. description

RDF Class: dcat:Catalog
Definition: A curated collection of metadata about resources (e.g., datasets and data services in the context of a data catalog)
Usage note: A Web-based data catalog is typically represented as a single instance of this class.
Reqirement Level mandatory
Cardinality: 1..1
Differences to DCAT: In DCAT 2021 this is a subclass of dcat:Dataset

Property: homepage

RDF Property: foaf:homepage
Definition: A homepage of the catalog (a public Web document usually available in HTML).
Range: foaf:Document
Requirement Level: Manadatory
Cardinality: mandatory: 1..1
Usage note: foaf:homepage is an inverse functional property (IFP) which means that it MUST be unique and precisely identify the Web-page for the resource. This property indicates the canonical Web-page, which might be helpful in cases where there is more than one Web-page about the resource.
Differences to DCAT-AP: In DCAT-AP this property is only recommend

Property: title

RDF Property: dct:title
Definition: This property contains a name given to the Catalogue. It can be repeated in multiple languages.
Range: rdfs:Literal
Requirement Level: Manadatory
Cardinality: 1..n
Usage note: The title is expected in at least one of the languages: de, fr, it, en

Property: title

RDF Property: dct:title
Definition: This property contains a description of the Catalogue. It can be repeated in multiple languages.
Range: rdfs:Literal
Requirement Level: Manadatory
Cardinality: 1..n
Usage note: The title is expected in at least one of the languages: de, fr, it, en

Property: publisher

The publisher is the organization that makes the decision to publish the catalog and has the legal authority to do so.

RDF Property: dct:publisher
Definition: This property refers to an entity (organisation) responsible for making the Catalogue available.
Range: foaf:Agent
Requirement Level: Manadatory
Cardinality: 1..n
Usage note: The publisher is the organization that makes the decision to publish the catalog and has the legal authority to do so.

Property: dataset

RDF Property: dcat:dataset
Definition: This property refers to an entity (organisation) responsible for making the Catalogue available.
Range: dcat:Dataset
Requirement Level: Manadatory
Cardinality: 1..1
Usage note: The collection of datasets published as a catalog

Class: Agent

RDF Class: foaf:Agent
Definition: The legal publisher of a data catalog.
Requiremnt Level: Mandatory
Cardinality: 1..n
Usage note: Every catalog or dataset is supposed to have just one publisher.

Property: name

RDF Property: foaf:name
Definition: The name of an organizatiob.
Range: rdfs:Literal
Requirement Level: Manadatory
Cardinality: 1..n
Usage note: The name can be provided in different languages
Differences to DCAT-AP: In DCAT-AP this property is only recommend
